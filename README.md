# نبراس · Nibras

الموقع الرابع في مشروع **مداد (Medad)** — موقع تسجيل لنادي نبراس الصيفي للطلاب.
بُني بنفس نظام تصميم مواقع مداد (نفس الـ stack، الألوان، المكوّنات، اتجاه RTL، الخطوط،
والحركة) — تغيّرت الهوية فقط (الاسم، الشعار، الخطوط، الألوان المميِّزة، والنصوص).

## التشغيل

```bash
npm install
npm run dev      # http://localhost:3009
```

أو انقر مرتين على `تشغيل-الموقع.command`.

## الصفحات (3 خطوات)

1. **`/`** — التعريف: شعار + الشعار النصي (اكتشف – جرّب – انطلق) + نبذة + تفاصيل النادي
   (الفئة، التاريخ، الوقت، الرسوم، الموقع على خريطة) + زر **سجّل معنا**. الحركة بـ GSAP + Lenis
   (CDN) كما في مواقع مداد، مع بديل (IntersectionObserver / reduced-motion).
2. **`/register`** — التسجيل: النموذج الكامل (الاسم الرباعي، الهوية، جوّالات، اختيار المرحلة
   والصف المترابط، الحي، الموقع الاختياري، سؤال الحساسية المشروط) مع تحقّق عربي.
3. **`/confirmation`** — الاعتماد: رقم العضوية بخط `Nibrasfont` + التنبيهات + زرّا
   (تسجيل طالب آخر / العودة للرئيسية).

## تعديل النصوص

**كل** النصوص القابلة للتغيير في ملف واحد: [`content.ts`](content.ts) — الوصف، التفاصيل،
التنبيهات، نص ملاحظة الموقع، روابط التواصل، وإحداثيات الخريطة، وأساس رقم العضوية. عدّل من هنا
دون لمس المكوّنات.

## الخطوط

- `Nibrasfont` → العناصر الأساسية (العناوين، الشعار النصي، رقم العضوية، أزرار CTA).
- `Thmanyah Sans` (خط مداد للنصوص) → النصوص الثانوية (الفقرات، التسميات، التذييل).

تُحمّل عبر `@font-face` في [`app/globals.css`](app/globals.css) من `public/fonts/` — نفس
أسلوب مواقع مداد.

## الألوان

نفس نظام مداد (cream / ink) مع تبديل اللون المميِّز من الذهبي إلى **برتقالي نبراس** المستخرَج
من الشعار، مع أزرق/سماوي/أحمر كألوان ثانوية. القيم في
[`tailwind.config.ts`](tailwind.config.ts) و [`app/globals.css`](app/globals.css).

## ⚠️ قراران تم رفعهما (Flagged decisions)

### 1) ميزة تحديد الموقع («أضاحي»)
لا يوجد موقع «أضاحي» في مساحة العمل — لا يوجد أي كود geolocation في المواقع الثلاثة الموجودة.
لذلك بُنيت الميزة من جديد في [`components/LocationPicker.tsx`](components/LocationPicker.tsx)
باستخدام `navigator.geolocation` + معاينة خرائط Google مضمّنة (بدون مفتاح API). الإحداثيات
الافتراضية للنادي في `content.ts` كـ `// TODO` للاستبدال.

### 2) طبقة البيانات ورقم العضوية (وضع «يعمل الآن» + TODO)
لا توجد قاعدة بيانات موصولة لهذا الموقع بعد. السلوك الحالي:

- إن لم يكن `DATABASE_URL` مضبوطاً → يُحفظ التسجيل محلياً في `.data/` ويُولّد رقم العضوية من
  عدّاد محلي (للتطوير فقط).
- إن ضُبط `DATABASE_URL` / `DIRECT_URL` (PostgreSQL، نفس نمط موقع التحضير) → يُحفظ في قاعدة
  البيانات عبر Prisma ويكون رقم العضوية = الأساس + المعرّف التسلسلي.

**لتفعيل الحفظ الحقيقي:** انسخ `.env.example` إلى `.env`، اضبط الرابطين، ثم:

```bash
npm run db:push
```

رقم العضوية: `membership.base` في `content.ts` (افتراضياً 1000). المنطق في
[`lib/membership.ts`](lib/membership.ts).
